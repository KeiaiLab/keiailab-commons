# AGENTS — operator-commons

This document is the runbook for AI agents and human contributors working
in the `operator-commons` repository. Project-specific guidance below
augments any global development conventions.

## Repository character

- This repository ships a Go *library*. Downstream consumers import it via
  the standard Go module mechanism (`go get` + `go.mod`).
- There is no `cmd/main.go`; no executable binary is produced.
- Public API breaking changes require an ADR plus a downstream-consumer
  impact analysis (see [GOVERNANCE.md](GOVERNANCE.md)).
- Tests are unit-level only. End-to-end and envtest coverage belong to the
  downstream operator that integrates this library.

## Before opening a change

1. **Public API impact** — if a function or type signature changes, plan
   an ADR plus a downstream impact note in the PR body.
2. **`go mod tidy` drift** — `lefthook` pre-push fails on `go.mod` /
   `go.sum` drift. Run `go mod tidy` before pushing.
3. **Test coverage** — every change inside `pkg/<sub>` must add or update
   a unit test.
4. **Dependency hygiene** — when adding a dependency, document the
   provenance and license in the PR body. Apache-2.0 transitive-only is
   the charter rule.
5. **Apache-2.0 license-only** — zero AGPL / BUSL transitive dependency
   target (`docs/kb/adr/0001-charter.md`).

## Local quality gates

The local hook layer (`lefthook.yml`) enforces:

- L1 pre-commit: `gofmt`, `go vet`, `golangci-lint`, gitleaks, GHA-block
  guard, DCO + Conventional Commits in commit-msg.
- L2 pre-push: `go test ./... -race -short`, `govulncheck`, `go mod tidy`
  drift, `go-licenses`, `markdown-link-check`, README 4-lang drift.
- L3 Makefile: `make lint test verify` for ad-hoc runs.
- L4 PR review: the PR body must cite the verification commands and quote
  the passing output of the gates above.

## ADR location

[`docs/kb/adr/`](kb/adr/INDEX.md). Use the format described in the index
header (Nygard five sections).

## Common workflows

- **Adding a new common package** (e.g. `pkg/status`, `pkg/finalizer`):
  1. Open an issue or draft ADR explaining the rationale and the
     downstream adoption plan.
  2. Wait a 7-day comment window per [GOVERNANCE.md](GOVERNANCE.md).
  3. Implement, add unit tests with ≥ 85 % coverage.
  4. Submit the PR with the ADR in the same commit set.

- **Bug fix in a single package**:
  1. Reproduce with a failing unit test.
  2. Fix and verify the test passes.
  3. PR with 1+ maintainer LGTM, then a patch release tag.

- **Dependency upgrade**:
  1. Open a PR (Renovate / Dependabot autogenerated or manual).
  2. Run `go mod tidy` and check `go test ./...` + `govulncheck`.
  3. Merge and tag if downstream impact is bounded.

## Anti-patterns

- Putting consumer-specific business logic into the library. The library
  ships *common* helpers only.
- Adding a new package without ADR + downstream adoption analysis.
- Silently changing a public API signature without an ADR.
